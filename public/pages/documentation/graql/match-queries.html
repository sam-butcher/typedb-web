    Match Queries

    Graql Match Queries
    Edit me

    A match query will search the graph for any subgraphs that match the given patterns. A result is produced for each match found, containing any variables in the query. The results of the query can be modified with various modifiers.

    Variable Patterns

    A pattern is a pattern to match in the graph. Patterns can be combined into a disjunction (‘or’) and grouped together with curly braces. Patterns are separated by semicolons, each pattern is independent of the others. A variable pattern is a pattern describing properties of a particular concept. The variable pattern can optionally be bound to a variable or an ID.

    • Graql
    • Java
    match $x isa pokemon;
{
  $x id "Mew"
} or {
  ($x, $y);
  $y isa pokemon-type, id "water";
}
    qb.match(
  var("x").isa("pokemon"),
  or(
    var("x").id("Mew"),
    and(
      var().rel("x").rel("y"),
      var("y").isa("pokemon-type").id("water")
    )
  )
);

    Properties

    isa

    Match instances that have the given type.

    • Graql
    • Java
    match $x isa pokemon;
    qb.match(var("x").isa("pokemon"));

    id

    Match concepts that have an id which matches the predicate.

    • Graql
    • Java
    match $x id "Articuno";

    qb.match(var("x").id("Articuno"));

    value

    Match concepts that have a value. If a predicate is provided, the resource must match that predicate.

    • Graql
    • Java
    match $x value contains "lightning";

    qb.match(var("x").value(contains("lightning")))

    has

    Match concepts that have a resource of type. If a predicate is provided, the resource must match that predicate.

    • Graql
    • Java
    match $x has pokedex-no < 20;

    qb.match(var("x").has("pokedex-no", lt(20)));

    The above is equivalent to:

    • Graql
    • Java
    match
(has-pokedex-no-owner: $x, has-pokedex-no-value: $pokedex-no) isa has-pokedex-no;
$pokedex-no value < 20;

    qb.match(
  var().rel("pokedex-no-owner", "x").rel("pokedex-no-value", "pokedex-no")
    .isa("has-pokedex-no"),
  var("pokedex-no").value(lt(20))
);

    relation

    Match concepts that are relations between the given variables. If a role type is provided, the role player must be playing that role type.

    • Graql
    • Java
    match
$x isa pokemon;
(ancestor: $x, $y);

    qb.match(
  var("x").isa("pokemon"),
  var().rel("ancestor", "x").rel("y")
);

    Type Properties

    The following properties only apply to types.

    sub

    Match types that are a subclass of the given type.

    • Graql
    • Java
    match $x sub type;

    qb.match(var("x").sub("type"))

    has-role

    Match relation types that have the given role.

    • Graql
    • Java
    match evolution has-role $x;

    qb.match(id("evolution").hasRole(var("x")));

    plays-role

    Match concept types that play the given role.

    • Graql
    • Java
    match $x plays-role ancestor;

    qb.match(var("x").playsRole("ancestor"));

    has-resource

    Match concept types that can have the given resource types.

    • Graql
    • Java
    match $x has-resource name;

    qb.match(var("x").hasResource("name"));

    The above is equivalent to:

    • Graql
    • Java
    match $x plays-role name-owner;

    qb.match(var("x").playsRole("name-owner"));

    Predicates

    A predicate is a boolean function applied to values. If a concept doesn’t have a value, all predicates are considered false.

    Comparators

    There are several standard comparators, =, !=, >, >=, < and <=. For longs and doubles, these sort by value. Strings are ordered lexicographically.

    • Graql
    • Java
    match $x has height = 19, has weight > 1500;

    qb.match(var("x").has("height", 19).has("weight", gt(1500)));

    If a concept doesn’t have a value, all predicates are considered false. The query below matches everything where the predicate >10 is true. So, it will find all concepts with value greater than 10. However, if a concept does not have a value at all, the predicate is considered false, so it won’t appear in the results.

    match $x value >10;

    Contains

    Asks if the given string is a substring.

    • Graql
    • Java
    match $x has description $desc;
$desc value contains "underground";


    qb.match(
    var("x").has("description", var("desc")),
    var("desc").value(contains("underground"))
);

    Regex

    Checks if the value matches a regular expression. This match is across the entire string, so if you want to match something within a string, you must surround the expression with .*.

    • Graql
    • Java
    match $x value /.*(fast|quick).*/;

    qb.match(var("x").value(regex(".*(fast|quick).*")));

    And and Or

    and and or allows combining predicates using boolean logic.

    • Graql
    • Java
    match $x has weight >20 and <30;

    qb.match(var("x").has("weight", gt(20).and(lt(30))));

    Modifiers

    • Graql
    • Java
    match $x isa pokemon, has pokedex-no $no;
select $x;
limit 30; offset 10; distinct; order by $no asc;


    qb.match(var("x").isa("pokemon").has("pokedex-no", var("no")))
    .select("x")
    .limit(30)
    .offset(10)
    .distinct()
    .orderBy("no", true);

    limit

    Limit the query to the given number of results.

    offset

    Offset the query by the given number of results.

    distinct

    Remove any duplicate results.

    order

    Order the results by the given variable’s degree. If a type is provided, order by the resource of that type on that concept. Order is ascending by default.

    select

    Indicates which variables to include in the results.
